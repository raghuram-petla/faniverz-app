# Cache Invalidation Audit

Audit all TanStack Query mutations and queries in the admin panel to find cache invalidation gaps — places where a mutation modifies data that affects a query's results, but doesn't invalidate that query's key.

## Steps

1. **Inventory all query keys**: Scan `admin/src/hooks/` for every `useQuery` / `useInfiniteQuery` call. Record each query key and what data it fetches.

2. **Inventory all mutations**: Scan `admin/src/hooks/` for every `useMutation` call (including those generated by `createCrudHooks` and `createMovieChildHooks` factories). Record what table/data each mutation modifies and which query keys it invalidates in `onSuccess`.

3. **Cross-reference for gaps**: For each mutation, determine ALL queries whose results could change because of that mutation. Check if the mutation invalidates those query keys. Note: TanStack Query uses **prefix matching** by default (no `exact: true`), so `invalidateQueries({ queryKey: ['admin', 'movies'] })` WILL match `['admin', 'movies', search, filter]`.

4. **Check composite save handlers**: Examine `useMovieEditHandlers.ts` (`handleSubmit`) and `useMovieAddState.ts` (`handleSubmit`) which fire multiple mutations. Verify they invalidate all cross-cutting query keys (theater views, platform-movie-ids, etc.).

5. **Report findings**: For each gap found, report:
   - **File and hook name**
   - **What data is modified**
   - **What query key is NOT invalidated but should be**
   - **User-visible symptom** (what stale data would the user see?)
   - **Severity**: CRITICAL (wrong data displayed) / HIGH (stale until refresh) / MEDIUM (stale for staleTime duration)

6. **Fix all gaps**: Apply the minimal fix for each gap — add missing `invalidateQueries` calls. Prefer using the `extraInvalidateKeys` option on `createMovieChildHooks` when applicable.

7. **Run quality gates**: `cd admin && npx tsc --noEmit && npx vitest run`

## Key patterns to watch for

- Mutations on junction tables (movie_production_houses, movie_platforms) that affect scoped queries
- Child mutations (theatrical runs, posters) that affect parent movie display or cross-page views
- The `['admin', 'platform-movie-ids']` prefetch cache used by useAdminMovies
- Theater-related query keys: `theater-movies`, `upcoming-movies`, `upcoming-rereleases`, `theater-search`
- `setMainPoster` which updates both `movie_posters.is_main` AND `movies.poster_url`
